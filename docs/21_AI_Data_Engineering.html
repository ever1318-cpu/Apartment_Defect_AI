21. AI Data Engineering

Sprint 1–6 재현 가능한 데이터 파이프라인

목표

원천 데이터 불변성, 그룹 누수 방지, 스키마 검증, 콘텐츠 해시 기반 버전 관리를 통해 대규모 하자 이미지 데이터셋을 재현 가능하게 구성한다.

파이프라인

Legacy CSV → Canonical JSONL → Validation → Group-stratified split
           → Version manifest → Training augmentation
           → Uncertainty sampling → Human review

Sprint 1–6 산출물

Sprint구현
1ImageRecord 모델, 원자적 JSONL 입출력, JSON Schema
2Legacy CSV 변환과 중복 식별
3레코드·파일·그룹 누수 검증
4결정적 그룹 계층 분할과 pytest
5SHA-256 데이터셋 버전 매니페스트
6능동학습 후보 선택과 결정적 증강 정책

CLI 예시

apartment-data import-legacy legacy.csv records.jsonl
apartment-data validate records.jsonl --root .
apartment-data split records.jsonl split-records.jsonl --seed 42
apartment-data manifest split-records.jsonl manifest.jsonl --version 2026.07.20
